Introduction

PHP is a popular programming language used for designing web pages. It is an open source scripting language that is executed on the Server with the result returned to the brwowser as plain HTML. It is free to use and download. PHP files include:

  • Text, HTML, CSS, JavaScript and PHP Code
  • PHP Code is executed on the server and the result is returned to the browser
  • PHP files are saved with the ".php" extension.

PHP can be used for web development or as a programming language (create scripts), to covert file from one format to another, transform images, create thumbnails or perform calculations. There is alot that can be done with PHP and this is what makes it a popular programming language.

Hello World

The best way to learn PHP is to create a small program that outputs Hello World. PHP code is embedded between the <?php and ?> tags. The PHP tags can be placed in the body of the HTML page The following code illustrates a simple PHP file that prints Hello World:

<html>
<head>PHP Code</head>
<body>
<?php
echo "Hello World";
?>
</body>
</html>

The key word echo is a built in function that PHP uses to display information to the user.

Variables and Types

Variables are containers that store different types of information. In PHP, all variables begin with the $ sign, which is followed by the name of the variable. PHP supports variables of type:

  • boolean
    • < li>float(real numbers)
  • integer(whole numbers)
  • string

Variables are declared in PHP using the following syntax:
$a=True;
$b=4.0;
$c=10;
$d="Hello World";

In the syntax above, four variables are defined. The first variable a is a boolean with value True, b is a float with value 4.0, c is an integerwith value 10 and d is a string with value Hello World. Numbers are declared as is while strings must be enclosed in single quotes or double quotes.

Simple Arrays

Arrays are variables that can hold more than one variable and displays them in list form. Arrays in PHP are created using the following syntax:

$number=array(1, 2. 3, 4);

Before any item in the list can be used, the array must be created. Items in an array are accessed by their index number. PHP uses zero based induces so the first item in an array will be at index 0 and to access it you would use the following syntax:

$array_name = [1, 2, 3]; (number array)
$array_name=["cats", "dogs", "horses"]; (string array)

To extract an item in one of the arrays above you use the following syntax:

$array_name[index];

Arrays with Keys

Arrays that are created as single lists allow items in the array to be accessed through their index. In arrays with keys, the items in the array are accessed through a key. Arrays in PHP are ordered maps where the map is a type that associates values to keys. An array with keys will be represented with the following syntax:

$student_id = [
"Peter"=> "00011",
'Don'=> "22234",
];

If we desire to get the student id of Don we will use the following syntax:

$student_id["Don"];

Multidimentional Arrays

Multiple dimentional arrays are arrays that contain one or more arrays. This type of array stores another array at each index in place of a single element. In other words, each element in the array can be an array and hold sub arrays.

Dimentions indicate the number of indices needed to select an element. A two dimentional array will require two indices to select an element while a three dimentional array will require three indices to select an element. Below is an exampple of a two dimentional array:

$multiarray = array(
          array(1, 2),
          array(3, 4)
        );

Below is the breakdown of this array :


        Array
         (
           [0]=> Array
             (
               [0]=>1
               
               [1]=>2
            )
           
          [1]=> Array
           (
            [0]=>3
            
            [1]=>4
           )
        
       )
To access the number 2, we use $multiarray[0][1], while if we want to access the number 3, we use $multiarray[1][0].

For Loops

For loops iterate through a block of code a specific number of times using an index. PHP has two types of For Loops, a simple type of for loop and a foreach loop. For loops are useful when we need to iterate over an array. The members of an array can be referred to using an index which changes with each iteratio. A simple for loop can be declared using the following syntax:

$numbers = array(0. 1, 2, 3, 4, 5];
for($i = 0; $i < count($numbers); $i++ ) {
$number = $number[$i];
echo $number."\n";
}

The code above includes the initial statement of the for loop which defines 3 parts:
$i =0; which initializes the loop.
$i < count($number); which is the conditional statemment. The loop will iterate as long as this is true;
$i++; which is the incremental statement of the loop.This can also be written as $i+=1;or $i = $i +1;

The foreach loop is used to loop through each key/value pair in array. The following is an example of a foreach loop:

$age = array(
Peter => 25,
Mike => 18,
Sarah => 30
);

foreach($age as $person=> $value) {
echo "$person=>$value;
}

While Loops

While loops are a block of code that executes as long as the condition specified in the loop remains true. For example, a function can be defined to perform mathematical operations or maniupulate strings or characters, or objects.

There are two types of while loops in PHP. The while loop that executes only when the specified condition is true and the do while loop that executes at least once and is repeated as long as that condition is true. The following examples ilustrate the syntax of the while loop and the do while loop:

/*** while loop***/
$x = 0;
while($x < 10) {
echo $x;
$x++;
}

/***do while loop***/
$x =0;
do {
echo $x;
$++;
} while($ < 10)

Functions

Functions are blocks of code that can be executed to perform operations on data or information in a program. The list below describles more detail about functions:

  • The blocks of statements in a function can be used repeatedly in a program
  • Functions are executed when they are called
  • Functions recieve a list of arguments separated by commas
  • May or not return a value using the return statment

PHP contains two types of functions: built in functions((over 1000) and user defined functions. The syntax for user defined functions is:

function nameFunction() {
Execute your code here;
}


The name of the function should represent what the function does. The following example illustretes a function that adds two numbers:

<?php
functipn addNum(int $x, int $y) {
return $x + $y;
}
echo addNum(3, 4);
? >

The function addNum adds two integers. It has two paremeters $x and $y. The function is called by the echo statement and the parementers are replaced by integers. When parameters are replaced by values when the function is called they become arguments.

When passing arguments in a funtion call, make sure they match the order and typ of the parameters in the function definition.

A complete listing of PHP built in functions is available in PHP Reference documentation. One of these functions is the array() function which is used in some of the examples in this document.

Objects

PHP is an object oriented language but many of its functins are not object oriented. Despite it object oriented nature, it does not have to be used as an object oriented language. Object oriented programming languages like PHP are based on the concepts of Objects, which contain specific properties and methods.

Properties are attributes or data that describe or provides us with specific details concerning an aspect of the object. The methods are operations that can be performed on the object. The interesting feature of an object in Object Oriented Programming is that its own methods can access and modify its properties or data.

A class is a template or container of an object so objects are instances of a class. In other words, a class is definition of an object and objects are instances of a class. To define a class, use the keyword class followed by the name of the class. The name is followed by a pair of curly braces with the property and methods of the class defined in this class. The example below illustrates this:

<?php
class Vegetables {
// properties
public $name;

//methods
function setName($name){
this->name=$name;
}
function getName(){
return $this->name;
}

// instatiate object
$corn = new Vegetable();
corn->setName("Corn");
echo $corn->getName();
?>
}

Exceptions

PHP exceptions are objects that describe errors or unexplained behaviour in the php script. This object contains information about the unexpected error or unexpected behaviour encountered by the function.

An exception can be thrown(throw) and caught(catch); each throy must have at least one corresponding catch. Exceptions can be thrown by

  • PHP Functions
  • User defined functions
  • Classes


A throw statement allows methods or user defined functions to throw an exception. Statements following the throw statement will not be executed once an exception is thrown. Once a throw is encountered, PHP will look for a matching catch block.

In a block of code with no catch statement, the exception is not caught. If an exception is not caught, a fatal error occurs with an Uncaught Exception message. A try catch statement will look as presented below: 
         
           try {
           code to throw exeption goes here
           } catch(Exception $e) {
          Run code when exception is caught here
    }
Catch blocks indicate the type of expception that should be caught and the variable used to access the exceptions. In this example, the type of exception is Exception and the variable used to access it is $e.There are different classes of exceptions and multiple catch blocks can be used to catch them.

PHP 5.5 introduced a finally block to specified after or instead of the catch blocks. Code within these blocks always execute(even if an exception has not been caught). The presence of this block makes the catch block optional. The syntax for this is as folows: 
        
	  try {
          code to throw exceptions
          }catch(Exception $e) {
          run when eception is caught
         }finally {
         Always run whether expception caught
       }<
       /code>
